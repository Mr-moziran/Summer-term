- 当前分支：`main`。

- 前端以远程仓库版本为准，当前没有本地前端改动。

详细工程约定见根目录 `AGENTS.md`。

## 2. 已完成的功能

### 知识库与自助问答

- `demo/src/main/resources/db/knowledge-base.json` 已有 **60 条**中文 SaaS FAQ。
- FAQ 覆盖账号、成员、套餐订阅、账单退款、权限、附件、导出、数据保留、投诉和功能建议等场景。
- 业务规则基准：`docs/data/业务规则基准.md`。
- FAQ 评测集：`docs/data/知识库评测集.json`。
- `KnowledgeBaseSeeder` 可将 FAQ 写入 pgvector；`ResolvedTicketSeedIndexer` 可将已解决/已关闭工单写入 pgvector。
- 向量元数据已隔离：FAQ 使用 `documentType=knowledge`，历史案例使用 `documentType=resolved-ticket`，两类检索不会互相干扰。

### 非业务问题拦截

- 已合并远程仓库的意图识别实现：`AskIntentClassifier`、`LocalAskIntentClassifier`、`DeepSeekAskIntentClassifier`。
- `AskAiService` 还会优先通过 `BusinessScopeChecker` 做快速规则拦截。
- 天气、新闻、股票、笑话等非业务问题返回：

  > 我们目前不能回答此类问题，请问还有其他业务相关问题需要帮助吗？

- 此类结果类型为 `OUT_OF_SCOPE`，不会检索知识库，也不会显示转人工/创建工单入口。

### 已验证的效果

- FAQ 检索：退款、附件、数据保留、自动续费、密码重置等问题能返回对应知识库内容。
- 非业务问题“今天天气怎么样？”返回 `OUT_OF_SCOPE`，不创建工单。
- Maven 测试曾执行成功：`mvn clean test`。

## 3. Kaggle 工单扩充（本轮最新进度）

### 数据来源与安全处理

- 来源：Kaggle 公开数据集 `suraj520/customer-support-ticket-dataset`，本地原始 CSV 共 **8,469** 条。
- 原始文件位置：`data/raw/customer-support-ticket-dataset/customer_support_tickets.csv`。
- 原始数据及处理后的候选文件被 `.gitignore` 忽略，**不得提交**。
- 清洗脚本：`tools/prepare-kaggle-tickets.ps1`。
- 脚本按工单类型各抽 100 条，生成 **500 条**候选记录到 `data/processed/kaggle-ticket-candidates.csv`。
- 已验证候选文件中标准邮箱、网址和混淆邮箱匹配数均为 0；脚本会移除个人字段，并对正文中的邮箱、网址、手机号脱敏。

### 场景改写与数据库导入

- 不直接导入英文电商工单，也不把它们当 FAQ。
- 已新增 `demo/src/main/resources/db/kaggle-curated-tickets.sql`：
  - 25 条中文 SaaS 场景的历史解决案例；
  - 25 条对应客服回复；
  - 案例只保留公开数据的“问题类型分布”，内容及规则按本项目 `业务规则基准.md` 重写。
- 该 SQL 已导入本地 PostgreSQL，且已核验：**25 条工单（13 RESOLVED、12 CLOSED）+ 25 条回复**。
- SQL 可重复执行，按工单标题和回复内容去重。
- 数据清洗与映射说明：`docs/data/Kaggle工单清洗与映射.md`。

## 4. 当前必须完成的一步

新导入的 25 条案例还没有写入向量库，因此 AI 的“相似工单检索”暂时不能命中它们。

在配置了有效 `EMBEDDING_API_KEY` 的 PowerShell 中执行：

```powershell
cd "D:\Machine_Learning\Summer term\demo"
mvn spring-boot:run "-Dspring-boot.run.arguments=--app.ai.index.seed.enabled=true"
```

说明：

- 此命令会重建 `resolved-ticket` 向量索引；完成启动日志后可用 `Ctrl+C` 关闭服务。
- 该操作需要 embedding API，当前助手无法读取用户终端中临时设置的密钥。
- 完成后可用以下 SQL 检查：

```sql
SELECT metadata->>'documentType' AS document_type, COUNT(*)
FROM vector_store
GROUP BY metadata->>'documentType'
ORDER BY document_type;
```

预期会看到 `knowledge` 和 `resolved-ticket` 两类记录；`knowledge` 应为 60。历史案例数量会随数据库中所有 `RESOLVED` / `CLOSED` 工单总数而变化。

## 5. 本地数据库与启动方式

- Docker 容器：`pgvector`。
- 数据库：`ticket_system`，端口 `5432`，用户名/密码 `postgres/postgres`。
- 初始化和全量种子导入命令已更新在根目录 `数据库启动.md`。
- `application.yml` 默认使用 DeepSeek + pgvector，但两个 `seed.enabled` 默认均为 `false`，避免每次启动重复重建向量。
