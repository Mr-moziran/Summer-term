# 知识库评测说明

`知识库评测集.json` 用于每次修改 FAQ、Embedding 模型、检索阈值或意图识别逻辑后进行回归验证。

## 判定规则

- `ANSWERED` 或 `ANSWERED_WITH_WARNING`：首条参考资料应为 `expectedReferenceTitle`，且不应创建工单。
- `ESCALATED`：必须创建 `PENDING` 工单；若指定了 `expectedReferenceTitle`，引用应包含该资料。
- `OUT_OF_SCOPE`：不得有知识库引用、不得创建工单，且 `canEscalate=false`。

## 执行方式

1. 使用普通用户账号登录前端，进入“智能助手”。
2. 逐条输入评测问题，并记录 `resultType`、首条参考资料、相似度和是否创建工单。
3. 不符合预期时，先检查业务规则和 FAQ 正文是否一致，再检查向量导入、Embedding 模型和阈值。
4. 对会创建工单的评测项，可在每轮评测后标记为测试数据或在演示数据库重置时清理。
