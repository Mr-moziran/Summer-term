# 前端开发手册

## 快速开始

### 1. 安装 Node.js

访问 [Node.js 官网](https://nodejs.org/) 下载并安装最新的 LTS 版本（推荐 v18 或 v20）。

安装完成后，在终端中验证：

```bash
node --version
npm --version
```

### 2. 安装项目依赖

在 `frontend` 目录下运行：

```bash
cd frontend
npm install
```

### 3. 启动开发服务器

```bash
npm run dev
```

浏览器访问 http://localhost:5173

## Mock 数据开发

在后端接口未完成前，可以使用 Mock 数据进行开发：

### 方式一：使用 Apifox Mock Server

1. 将接口文档导入 Apifox
2. 开启 Mock 服务
3. 修改 `vite.config.js` 中的代理地址

```javascript
proxy: {
  '/api': {
    target: 'https://mock.apifox.cn/m1/xxxxx',  // Apifox Mock 地址
    changeOrigin: true,
    rewrite: (path) => path.replace(/^\/api/, '')
  }
}
```

### 方式二：本地 Mock 数据

在 `src/api/` 目录下，可以暂时返回假数据：

```javascript
// 示例
export function getTickets(params) {
  // return request({ url: '/tickets', method: 'GET', params })
  
  // 临时返回 Mock 数据
  return Promise.resolve({
    content: [
      { id: 1, title: '测试工单', status: 'PENDING', ... }
    ],
    totalElements: 1
  })
}
```

## 路由说明

### 角色权限

| 路由路径 | 角色 | 说明 |
|---------|------|------|
| `/login` | 公开 | 登录页 |
| `/register` | 公开 | 注册页 |
| `/my-tickets` | USER | 我的工单列表 |
| `/tickets/new` | USER | 提交工单 |
| `/tickets/:id` | USER | 工单详情 |
| `/agent/tickets` | AGENT | 客服工单列表 |
| `/agent/tickets/:id` | AGENT | 处理工单 |
| `/admin/dashboard` | ADMIN | 数据大盘 |
| `/admin/tickets` | ADMIN | 工单管理 |
| `/admin/users` | ADMIN | 用户管理 |

### 路由守卫

在 `router/index.js` 中已配置：
- 未登录用户访问受保护页面 → 跳转登录页
- 已登录用户访问登录页 → 跳转对应首页
- 角色不匹配 → 跳转登录页

## 状态管理

### User Store

```javascript
import { useUserStore } from '@/store/user'

const userStore = useUserStore()

// 获取状态
userStore.isLoggedIn    // 是否登录
userStore.username      // 用户名
userStore.role          // 角色：USER / AGENT / ADMIN
userStore.isUser        // 是否普通用户
userStore.isAgent       // 是否客服
userStore.isAdmin       // 是否管理员

// 调用方法
await userStore.loginAction({ email, password })
await userStore.registerAction({ username, email, password })
userStore.logout()
```

### Notification Store

```javascript
import { useNotificationStore } from '@/store/notification'

const notificationStore = useNotificationStore()

// 获取状态
notificationStore.notifications   // 通知列表
notificationStore.unreadCount    // 未读数量
notificationStore.hasUnread      // 是否有未读

// 调用方法
notificationStore.connect()      // 连接 WebSocket
notificationStore.disconnect()   // 断开连接
notificationStore.markAsRead(id) // 标记已读
notificationStore.markAllAsRead() // 全部已读
```

## API 调用示例

```javascript
import { getTickets, createTicket } from '@/api/ticket'

// 获取工单列表
const data = await getTickets({
  status: 'PENDING',
  page: 0,
  size: 10
})

// 创建工单
await createTicket({
  title: '无法登录',
  description: '密码错误'
})
```

所有 API 调用都经过 `utils/request.js` 的拦截器处理：
- 自动添加 Token
- 统一错误处理
- 401 自动跳转登录页

## WebSocket 通知

WebSocket 连接在用户登录后自动建立：

```javascript
// App.vue 中已处理
onMounted(() => {
  if (userStore.isLoggedIn) {
    notificationStore.connect()
  }
})
```

接收到通知时会：
1. 显示桌面通知（Element Plus Notification）
2. 更新通知列表
3. 增加未读计数

## 常见问题

### Q: npm install 速度慢？
A: 使用淘宝镜像源
```bash
npm config set registry https://registry.npmmirror.com
```

### Q: 端口被占用？
A: 修改 `vite.config.js` 中的端口号
```javascript
server: {
  port: 5174  // 改为其他端口
}
```

### Q: 如何查看 API 请求？
A: 打开浏览器开发者工具 → Network 标签

### Q: Token 存储在哪里？
A: localStorage，key 为 `token` 和 `userInfo`
