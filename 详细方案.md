# 企业智能客服工单系统 — 详细设计方案

---

## 一、项目背景与目

### 背景

企业客服部门每日处理大量用户工单，人工处理存在三个核心痛点：

- **分类慢**：人工判断工单类型需要阅读全文，耗时且标准不统一
- **回复慢**：客服需要翻阅历史记录寻找类似问题的解决方案
- **数据沉淀差**：历史工单的知识没有被有效复用

### 目标

引入大语言模型（LLM），在不改变现有客服工作流程的前提下，通过 AI 辅助将简单工单的处理时间从 5 分钟压缩至 1 分钟以内，同时沉淀历史知识供后续检索复用。

│  │       AI 服务层           │                   │
│  │  分类器  │  RAG检索  │ 生成器│                 │

## 五、数据库设计

### 核心表结构

**users（用户表）**

```sql
id, username, password_hash, email, role(USER/AGENT/ADMIN), 
created_at, is_active
```

**tickets（工单表）**

```sql
id, title, description, status, priority(LOW/MEDIUM/HIGH/URGENT),
category(TECH/BILLING/COMPLAINT/OTHER), 
submitter_id, assignee_id, 
created_at, updated_at, closed_at
```

**ticket_replies（回复表）**

```sql
id, ticket_id, author_id, content, is_ai_suggested, 
adopted_ai_suggestion, created_at
```

**ticket_vectors（向量表，pgvector）**

```sql
id, ticket_id, content_summary, embedding vector(1536), 
resolution TEXT, created_at
```

**notifications（通知表）**

```sql
id, user_id, ticket_id, message, is_read, created_at
```

## 六、AI 模块设计

### 6.1 工单提交时 — 自动分类与优先级判断

**触发时机**：用户提交工单后异步执行（不阻塞提交响应）

**Prompt 设计**：

```
你是一个客服工单分类助手。请根据以下工单内容完成两项判断：
1. 类别：从 [技术问题, 账单问题, 投诉, 其他] 中选一个
2. 优先级：从 [低, 中, 高, 紧急] 中选一个，紧急仅用于系统故障或资金损失

工单标题：{title}
工单内容：{description}

请以 JSON 格式返回：{"category": "...", "priority": "...", "reason": "..."}
```

**处理流程**：

```
用户提交工单
     │
     ▼
异步任务队列
     │
     ▼
调用 DeepSeek API（结构化输出）
     │
     ▼
解析 JSON → 更新工单 category/priority 字段
     │
     ▼
同步将工单文本向量化 → 存入 ticket_vectors
```

### 6.2 客服处理时 — RAG 辅助回复生成

**触发时机**：客服打开工单详情页时触发

**RAG 流程**：

```
工单内容
   │
   ▼
调用嵌入模型生成查询向量
   │
   ▼
pgvector 余弦相似度检索 Top-3 历史工单
   │
   ▼
拼装 Prompt（工单内容 + 3条历史解决方案）
   │
   ▼
调用 DeepSeek 生成回复建议
   │
   ▼
SSE 流式返回给前端（打字机效果）
```

**Prompt 模板**：

```
你是一名专业客服，请根据当前工单和历史参考案例，生成一份回复草稿。

【当前工单】
{ticket_content}

【历史相似案例】
案例1：{similar_case_1}
案例2：{similar_case_2}
案例3：{similar_case_3}

要求：语气专业友好，给出具体解决步骤，如无法解决请说明升级流程。
```

## 九、技术难点与解决方案

### 难点1：RAG 检索质量（核心难点）

**问题**：历史工单数量少时，检索结果质量差，AI 建议无参考价值。

**解决**：

1. 预置 50-100 条模拟历史工单数据（涵盖各类别）作为知识库基础
2. 向量化时将工单标题 + 描述 + 解决方案拼接后再嵌入，保留更多语义信息
3. 相似度阈值过滤（低于 0.75 的结果不传入 Prompt）

### 难点2：Prompt 工程

**问题**：LLM 生成的回复可能过于通用，实际价值低。

**解决**：

1. 在 Prompt 中明确输出格式（问题确认 → 解决步骤 → 结束语）
2. 对不同工单类别使用不同的 Prompt 模板（技术类 vs 账单类）
3. 设置 temperature=0.3，降低随机性，保证回复稳定

### 难点3：LLM 并发调用

**问题**：多个客服同时打开工单触发 AI，可能超出 API 速率限制。

**解决**：AI 辅助生成接口使用异步执行（`@Async`），结合 SSE 流式返回，前端展示加载动画，避免接口超时。
