# 本地 PostgreSQL + PgVector 初始化

项目默认连接 `localhost:5432/ticket_system`，用户名和密码均为 `postgres`。

## 1. 启动数据库

```powershell
docker pull pgvector/pgvector:0.8.3-pg18-trixie
docker run -d --name pgvector -p 5432:5432 -e POSTGRES_PASSWORD=postgres -e POSTGRES_DB=ticket_system pgvector/pgvector:0.8.3-pg18-trixie
```

若容器已存在，使用：

```powershell
docker start pgvector
```

## 2. 导入业务表与演示数据

应用默认不自动执行 SQL 初始化。首次启动前，在项目根目录执行：

```powershell
docker cp demo/src/main/resources/db/schema.sql pgvector:/tmp/ticket-system-schema.sql
docker cp demo/src/main/resources/db/seed.sql pgvector:/tmp/ticket-system-seed.sql
docker cp demo/src/main/resources/db/kaggle-curated-tickets.sql pgvector:/tmp/kaggle-curated-tickets.sql
docker exec pgvector psql -v ON_ERROR_STOP=1 -U postgres -d ticket_system -f /tmp/ticket-system-schema.sql
docker exec pgvector psql -v ON_ERROR_STOP=1 -U postgres -d ticket_system -f /tmp/ticket-system-seed.sql
docker exec pgvector psql -v ON_ERROR_STOP=1 -U postgres -d ticket_system -f /tmp/kaggle-curated-tickets.sql
```

`seed.sql` 和 `kaggle-curated-tickets.sql` 都可以重复执行：会补齐演示账号、18 张基础工单以及 50 条已脱敏、按 SaaS 规则改写的历史解决案例，不会重复写入相同标题的工单。

## 3. 初始化向量知识库与历史案例

先配置环境变量。知识库和相似案例使用 SiliconFlow 的 `BAAI/bge-m3` embedding，因此需要有效的 `EMBEDDING_API_KEY`；聊天模型仍可保持本地规则模式。

```powershell
$env:EMBEDDING_API_KEY = '你的 SiliconFlow API Key'
cd demo
mvn spring-boot:run "-Dspring-boot.run.arguments=--app.ai.ask.knowledge.provider=pgvector --app.ai.ask.knowledge.seed.enabled=true --app.ai.index.provider=pgvector --app.ai.index.seed.enabled=true"
```

这次启动会自动创建 Spring AI 的 `vector_store` 表，并执行两项可重复的导入：

- `knowledge-base.json` 中的 300 条中文 SaaS FAQ；
- 数据库中状态为 `RESOLVED` 或 `CLOSED` 的历史工单，用于相似工单检索。

完成后关闭应用，并在日常启动时保持两个 `seed.enabled` 为 `false`，仅保留：

```yaml
app:
  ai:
    ask:
      knowledge:
        provider: pgvector
    similarity:
      provider: pgvector
    index:
      provider: pgvector
```

## 4. 验证数据

```powershell
docker exec -it pgvector psql -U postgres -d ticket_system
```

进入 `psql` 后执行：

```sql
SELECT role, COUNT(*) FROM users GROUP BY role ORDER BY role;
SELECT status, COUNT(*) FROM tickets GROUP BY status ORDER BY status;
SELECT metadata->>'documentType' AS document_type, COUNT(*) FROM vector_store GROUP BY metadata->>'documentType';
```

最后一条应能看到 `knowledge` 和 `resolved-ticket` 两类文档。
